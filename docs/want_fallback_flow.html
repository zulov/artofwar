Want Fallback Flow

The AI checks whether a request is executable before adding it to the WantList. If the request needs a missing producer building, it emits that building instead and retries the original want on the next AI tick. The rule is one hop only.

Rule: one hop only. If the prerequisite is still not there next tick, the AI re-evaluates and can emit the same fallback again. Do not recurse further inside one action pass.

Summary

Requests that are directly executable keep their original priority. Requests that are impossible because the producer building does not exist are converted into a matching BUILDING want. Requests that are only blocked because the building exists but is not ready are dropped for this tick and can be re-issued next tick.
