Brain Input Refactor Proposal

Changes to inputs for all 5 brains + new BrainIndex enum for prefix indexing

0. BrainIndex Enum

Replace magic numbers in nation->actionPrefix[0], nation->actionPrefix[1], etc. with a typed enum.

// BrainIndex.h enum class ActionBrainIndex : unsigned char { MASTER = 0, ECONOMY = 1, BUILDING = 2, UNIT = 3, }; enum class OrderBrainIndex : unsigned char { MILITARY = 0, };

Usage: nation->actionPrefix[cast(ActionBrainIndex::MASTER)] + "master.csv"

Why the same 6 inputs as UnitBrain? Knowing army composition is critical for military decisions: a cavalry-heavy force should attack (fast raids), a range-heavy force should defend (positional advantage), and enemy composition determines whether engaging is favorable.
