Resource Building Types & Roles

Every resource building in the game has one or more capabilities along 5 dimensions: BONUS (speed boost near nodes), SOURCE (creates/renews resource nodes), STORAGE, and REFINE. A single building can have multiple capabilities (dual-purpose). The AI's EconomyBrain emits a need per capability+resource (10 NEED_* outputs) and matches them to buildings by these DB-derived capabilities.

Mechanism: Building Resource Bonuses

Passive Multiplier

Resource buildings with collect > 0 and resourceRange > 0 project a collection speed bonus onto all resource nodes within range. Workers collecting from a boosted node get the bonus multiplied into their gathered amount.

Effective Collection Formula
collected = unit.collect * building.collect_bonus
where bonus = max(all overlapping buildings' level.collect) for that resource type
default bonus (no building nearby) = 1.0

Buildings: mill (collect=1.25, resRng=15), sawmill (1.25, 15), mine_s (1.25, 15), mine_g (1.25, 15), tree_nursery (collect=0, resRng=10 -- marks cells as bonused but gives no speed bonus).

Mechanism: Stone & Gold Refinement

Passive / Every 1 second

Buildings with refine capacity generate bonus resources per second proportional to gathering rate, capped by total refine capacity, at 10% rate.

Refinement Formula (per second)
stoneBonus = min(stoneGatheredThisSecond, totalStoneRefineCapacity) * 0.10
goldBonus = min(goldGatheredThisSecond, totalGoldRefineCapacity) * 0.10

Buildings: stoneRefinery (stoneRefCap=10), goldRefinery (goldRefCap=10). Rates hardcoded at 0.10.

Mechanism: Gold Interest

Periodic / Every Year

Once per year, 1% interest on stored gold up to gold storage capacity.

Gold Interest Formula (per year)
goldInterest = min(goldStorage, currentGold) * 0.01

Gold storage providers: center (200), bank (200 each).

Mechanism: Ruinable Salvage

One-time / On Combat Death

When a building with ruinable=true is destroyed in combat (HP=0), it leaves a salvage resource node. Type is whichever of wood or stone the building cost more of. Amount is 20% of that cost. Salvage nodes have 2x collectSpeed.

Ruinable Formula
Resource type: moreWoodThanStone ? woodpile (id=6) : ruin (id=5)
Amount (node HP): maxFromWoodOrStone * 0.20

All buildings are ruinable=1 except farm (id=10, ruinable=0).

Mechanism: Resource Spawning

Active / Queue-based loop

Buildings with toResource ≥ 0 and spawnResourceRange > 0 periodically create new resource nodes nearby. When the queue is empty, it auto-queues RESOURCE_CREATE. After spawnResourceTime frames, one resource node is placed on the first valid empty cell within spawnResourceRange. Repeats infinitely.

Only building: tree_nursery (id=17). Spawns young_tree (id=7, 50HP wood, 1x1, collectSpeed=1.2) every 900 frames (~30s) within range 100.

Mechanism: toResource Self-Conversion

One-time / On Build Complete

Buildings with toResource ≥ 0 and spawnResourceRange ≤ 0 immediately self-destruct on completion and place a resource node of type toResource at their position.

Only building: farm (id=10, toResource=4). Converts to farm resource node (id=4, 1600HP food, 4x4, 12 maxUsers, collectSpeed=1.2).

Also triggers on peaceful demolition (HP > 0) for any building with toResource ≥ 0 (e.g. demolishing a tree_nursery yields one young_tree).

Mechanism: Food Decay

Loss / Every Month

Food stored beyond total foodStorage capacity decays by 10% each month. Only excess is affected.

Food Decay Formula (per month)
foodLost = max(0, currentFood - foodStorage) * 0.10

Food storage providers: center (200), house (20 each), granary (200).

AI Resource Building Decision Architecture

The AI decides which resource building to construct using a need-based system built into the unified EconomyBrain. There is no separate resource sub-brain. For the full brain I/O spec see economy_brain_architecture.html. The flow is:

Step 2: EconomyBrain Building-Need Outputs (10)

The unified EconomyBrain emits 10 direct building-need outputs (tanh, [-1,+1]). Each names a capability + resource rather than a specific building, so it stays nation-agnostic:

OutputCapabilityResourceMatches building where...
NEED_BONUS_FOODBONUSFOODcollect > 0, resourceRange > 0, resourceType == FOOD
NEED_BONUS_WOODBONUSWOODcollect > 0, resourceRange > 0, resourceType == WOOD
NEED_BONUS_STONEBONUSSTONEcollect > 0, resourceRange > 0, resourceType == STONE
NEED_BONUS_GOLDBONUSGOLDcollect > 0, resourceRange > 0, resourceType == GOLD
NEED_FOOD_SOURCESOURCEFOODtoResource >= 0, spawnResourceRange <= 0 (e.g. farm)
NEED_WOOD_SOURCESOURCEWOODtoResource >= 0, spawnResourceRange > 0 (e.g. tree nursery)
NEED_FOOD_STORAGESTORAGEFOODfoodStorage > 0 (e.g. granary)
NEED_GOLD_STORAGESTORAGEGOLDgoldStorage > 0 (e.g. bank)
NEED_STONE_REFINEREFINESTONEstoneRefineCapacity > 0
NEED_GOLD_REFINEREFINEGOLDgoldRefineCapacity > 0

These outputs are produced from the brain's resource-economy inputs: per-resource stockpiles & gather speeds, storage ratios, refine ratios/gaps, bonus coverage, nearby supply, resource-building counts, and lacking feedback. (Full input list in economy_brain_architecture.html.)

Step 3: Scoring & Selection (createResBuilding)

Each tick AiOrchestrator::createResBuilding() walks every resource building the nation can build and assigns it a single need = the max of whichever NEED_* outputs match its capabilities (by the DB fields above). The top 3 buildings with need > 0.1 are submitted to the WantList as BUILDING requests.

Per-building need (AiOrchestrator::createResBuilding)
need = 0
if FOOD bonus  (collect>0, resRange>0, resType==FOOD):  need = max(need, NEED_BONUS_FOOD)
if WOOD bonus / STONE bonus / GOLD bonus:               need = max(need, NEED_BONUS_*)
if food source (toResource>=0, spawnResourceRange<=0):  need = max(need, NEED_FOOD_SOURCE)
if wood source (toResource>=0, spawnResourceRange>0):   need = max(need, NEED_WOOD_SOURCE)
if foodStorage>0:                                       need = max(need, NEED_FOOD_STORAGE)
if goldStorage>0:                                       need = max(need, NEED_GOLD_STORAGE)
if stoneRefineCapacity>0:                               need = max(need, NEED_STONE_REFINE)
if goldRefineCapacity>0:                                need = max(need, NEED_GOLD_REFINE)

keep top 3 by need (need > 0.1) -> wantList.addRequest(BUILDING, need, building.id)

A building with several capabilities (e.g. a bank with BONUS+STORAGE, or a tree nursery with BONUS+SOURCE) takes the maximum across all the needs it satisfies. Resource-building upgrades use a separate weighted-choice path (resolveResBuildingUpgrade) driven by RES_BUILDING_UPGRADE_URGENCY plus resource-type priorities.

Placement Strategy

When the WantList executes a BUILDING want, WantExecutor::findPosToBuild() chooses where to place it based on the building's parent type:

Parent typeMethodLogic
RESOURCE BuildSpatialBrain → Environment::getPosToCreate() BuildSpatialBrain now carries a merged resource-not-bonus output, so resource-bonus buildings can still be placed through the brain without a separate resource-routing path.
OTHER / DEFENCE / TECH / UNITS BuildSpatialBrain → Environment::getPosToCreate() BuildSpatialBrain outputs 12 influence-map weights (buildings, units, attackSpeed, 4x gatherSpeed, 3x enemy, res-not-bonus, economy); the weighted maps are combined and the best visible cell is chosen.
