Resource Building Types & Roles

Every resource building in the game has one or more capabilities. The AI classifies each building into 5 capability dimensions: BONUS, SPAWN, CONVERT, STORAGE, REFINE. A single building can have multiple capabilities (dual-purpose).

Mechanism: Building Resource Bonuses

Passive Multiplier

Resource buildings with collect > 0 and resourceRange > 0 project a collection speed bonus onto all resource nodes within range. Workers collecting from a boosted node get the bonus multiplied into their gathered amount.

Effective Collection Formula
collected = unit.collect * building.collect_bonus
where bonus = max(all overlapping buildings' level.collect) for that resource type
default bonus (no building nearby) = 1.0

Buildings: mill (collect=1.25, resRng=15), sawmill (1.25, 15), mine_s (1.25, 15), mine_g (1.25, 15), tree_nursery (collect=0, resRng=10 -- marks cells as bonused but gives no speed bonus).

Mechanism: Stone & Gold Refinement

Passive / Every 1 second

Buildings with refine capacity generate bonus resources per second proportional to gathering rate, capped by total refine capacity, at 10% rate.

Refinement Formula (per second)
stoneBonus = min(stoneGatheredThisSecond, totalStoneRefineCapacity) * 0.10
goldBonus = min(goldGatheredThisSecond, totalGoldRefineCapacity) * 0.10

Buildings: stoneRefinery (stoneRefCap=10), goldRefinery (goldRefCap=10). Rates hardcoded at 0.10.

Mechanism: Gold Interest

Periodic / Every Year

Once per year, 1% interest on stored gold up to gold storage capacity.

Gold Interest Formula (per year)
goldInterest = min(goldStorage, currentGold) * 0.01

Gold storage providers: center (200), bank (200 each).

Mechanism: Ruinable Salvage

One-time / On Combat Death

When a building with ruinable=true is destroyed in combat (HP=0), it leaves a salvage resource node. Type is whichever of wood or stone the building cost more of. Amount is 20% of that cost. Salvage nodes have 2x collectSpeed.

Ruinable Formula
Resource type: moreWoodThanStone ? woodpile (id=6) : ruin (id=5)
Amount (node HP): maxFromWoodOrStone * 0.20

All buildings are ruinable=1 except farm (id=10, ruinable=0).

Mechanism: Resource Spawning

Active / Queue-based loop

Buildings with toResource ≥ 0 and spawnResourceRange > 0 periodically create new resource nodes nearby. When the queue is empty, it auto-queues RESOURCE_CREATE. After spawnResourceTime frames, one resource node is placed on the first valid empty cell within spawnResourceRange. Repeats infinitely.

Only building: tree_nursery (id=17). Spawns young_tree (id=7, 50HP wood, 1x1, collectSpeed=1.2) every 900 frames (~30s) within range 100.

Mechanism: toResource Self-Conversion

One-time / On Build Complete

Buildings with toResource ≥ 0 and spawnResourceRange ≤ 0 immediately self-destruct on completion and place a resource node of type toResource at their position.

Only building: farm (id=10, toResource=4). Converts to farm resource node (id=4, 1600HP food, 4x4, 12 maxUsers, collectSpeed=1.2).

Also triggers on peaceful demolition (HP > 0) for any building with toResource ≥ 0 (e.g. demolishing a tree_nursery yields one young_tree).

Mechanism: Food Decay

Loss / Every Month

Food stored beyond total foodStorage capacity decays by 10% each month. Only excess is affected.

Food Decay Formula (per month)
foodLost = max(0, currentFood - foodStorage) * 0.10

Food storage providers: center (200), house (20 each), granary (200).

Step 4: Building Capability Vectors & Scoring

Each candidate building gets a capability vector computed from its DB fields. The building with the highest dot product score = sum(need[i] * capability[i]) wins.

Placement Strategy per Building Capability

After the best building is chosen, findPosToBuild() dispatches to a placement method based on the building's primary capability:

CapabilityConditionMethodLogic
BONUS collect > 0 && resourceRange > 0 getPosToCreateResBonus() Uses resNotInBonus influence maps to find positions near unbonused resource nodes of matching type.
SPAWN toResource >= 0 && spawnRange > 0 getPosToCreateSpawn() Uses gather speed influence maps with equal weights across all 4 resource types. Places near active worker areas.
CONVERT toResource >= 0 && spawnRange <= 0 getPosToCreateConvert() Uses gather speed influence maps with 3x food emphasis (weights: food=3, wood=0.5, stone=0.5, gold=0.5). Places near food-gathering workers.
STORAGE / REFINE / other fallback getPosToCreateResBonus() Falls back to bonus placement near resource nodes of matching type.
ActionMaker.cpp: findPosToBuild() | Environment.cpp: getPosToCreateSpawn(), getPosToCreateConvert()
