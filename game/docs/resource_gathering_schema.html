2. Building Resource Bonuses

Passive Multiplier

Resource buildings (mills, lumber camps, mines) project a collection speed bonus onto all resource nodes within their resourceRange. When a worker collects from a boosted node, the bonus multiplies the gathered amount.

Effective Collection Formula

where bonus = max(all overlapping buildings' level.collect) for that resource type

3. Stone & Gold Refinement

Passive / Every 1 second

Buildings with refine capacity generate bonus resources per second, proportional to how much the player is actively gathering. The bonus is capped by the total refine capacity of all buildings, then multiplied by a 10% rate.

Refinement Formula (per second)
stoneBonus = min(stoneGatheredThisSecond, totalStoneRefineCapacity) * 0.10
goldBonus = min(goldGatheredThisSecond, totalGoldRefineCapacity) * 0.10

4. Gold Interest (Yearly Income)

Periodic / Every Year

Once per in-game year, the player earns 1% interest on their stored gold, up to their total gold storage capacity. This rewards accumulating gold and building storage structures.

Gold Interest Formula (per year)
goldInterest = min(goldStorage, currentGold) * 0.01

5a. Ruinable -- Salvage Ruins on Destruction

One-time / On Death (HP=0)

When a building with ruinable = true is destroyed in combat (HP reaches 0), it leaves behind a salvage resource node at its position. The resource type is whichever of wood or stone the building cost more of, and the amount is 20% of that cost.

Ruinable Formula
Trigger: building.HP reaches 0 (destroyed by damage)
Resource type: moreWoodThanStone ? WOOD_NODE(6) : STONE_NODE(5)
Amount (node HP): maxFromWoodOrStone * 0.20

5b. Resource Spawning -- Periodic Node Creation

Active / Queue-based loop

Buildings with both toResource > 0 and spawnResourceRange > 0 periodically create new resource nodes nearby. When the building's queue is empty, it auto-queues a RESOURCE_CREATE action. After spawnResourceTime seconds, one new resource node is placed on the first valid empty cell within spawnResourceRange. The cycle then repeats infinitely.

Spawn Conditions (all must be true)
1. building.toResource > 0   (resource type configured)
2. level.spawnResourceRange > 0   (spawn ability enabled at this level)
3. queue.isEmpty()   (not busy)

Duration: level.spawnResourceTime seconds
Result: 1 resource node of type toResource placed within spawnResourceRange

5c. toResource -- Building Self-Converts to Resource

One-time / On Peaceful Disposal

Buildings with toResource ≥ 0 but no spawn range (spawnResourceRange ≤ 0) are one-shot converters: upon becoming ALIVE, they immediately self-destruct and transform into a resource node of type toResource at their grid position. This also triggers when a building is peacefully demolished (HP > 0) if toResource ≥ 0.

Two Code Paths
Path A -- Auto-conversion (no spawn range):
State ALIVE: if toResource ≥ 0 AND spawnResourceRange ≤ 0   → immediately transition to DEAD → DISPOSE → place resource node

Path B -- Peaceful demolition:
State DISPOSE with HP > 0: if toResource ≥ 0   → place resource node of type toResource at building position

Path A: Auto-conversion (e.g., "Plant Field")

Player builds a "field" building (toResource=FOOD, spawnRange=0)

Immediate → DEAD → DISPOSE → Food node placed at building position

Path B: Peaceful demolition

Player manually demolishes a farm (toResource=FOOD, HP > 0)

DISPOSE state → HP > 0 (not destroyed by enemy)

toResource ≥ 0 → place Food node at building position

The building is peacefully removed and leaves behind a harvestable resource.

Key distinction from ruinable: toResource triggers on peaceful removal (HP > 0). ruinable triggers on combat destruction (HP = 0). They are mutually exclusive triggers in the DISPOSE state.

6. Food Decay (Resource Loss)

Loss / Every Month

Food stored beyond the player's total foodStorage capacity decays by 10% each month. Only the excess amount is affected. This creates pressure to build granaries/storage buildings to protect food reserves.

AI Decision Coverage -- Detailed Analysis

The AI uses a neural network architecture (Brain) with pre-trained CSV weight matrices. Two core classes drive decisions: ActionMaker (what to build) and OrderMaker (what orders to give). Each feeds metric vectors into brain networks that output scored decisions.

2. Building Bonus -- FULLY CONSIDERED

Brain: whichBuildingTypeResource

Input metrics: unbonused resource counts per type (weight 20), stockpiles (weight 1000)

Guard: Aborts if getResWithOutBonus() < 0.5 (no unbonused nodes exist)

Placement: getPosToCreateResBonus() uses influence maps to find positions near unbonused resource nodes

Files: ActionMaker.cpp:87-107, 191-208, 314-317

3. Refinement -- NOT CONSIDERED

No AI metric exists for stoneRefineCapacity or goldRefineCapacity.

Not in ResourceMetricIdx or BuildingMetricIdx.

AI may incidentally build buildings that have refine capacity, but has zero awareness of the refinement mechanic.

Impact: AI misses up to +10% bonus on stone/gold gathering.

4. Gold Interest -- NOT CONSIDERED

No AI metric for goldStorage, potentialGoldGain, or goldGainRate.

AI cannot strategically build storage for interest income.

Impact: Low -- only 1% per year. Rarely game-changing.

5a. Ruinable -- NOT CONSIDERED

ruinable field exists in db_building but is absent from all AI metrics.

AI cannot factor salvage value into attack/defense decisions.

Impact: Low -- passive effect, no decision needed.

5b. Resource Spawning -- PARTIAL

Dead code: ifFarmBuild brain is loaded from CSV but never called anywhere in the codebase.

Spawn buildings may be built incidentally because they match ParentBuildingType::RESOURCE category.

No metrics for spawnResourceTime, spawnResourceRange.

Impact: High -- farms are a key food source. AI should explicitly decide when to build them.

Files: ActionMaker.h:94 (loaded), never invoked

5c. toResource -- NOT CONSIDERED

toResource is not in any AI metric enum or input vector.

AI cannot evaluate one-shot resource conversion buildings.

Impact: Medium -- depends on game design reliance on this mechanic.

6. Food Decay -- NOT CONSIDERED

No AI metric for foodStorage, potentialFoodLost, or foodLostRate.

AI cannot build granaries to prevent food decay.

Impact: Medium-High -- AI can lose significant food reserves without knowing why.
